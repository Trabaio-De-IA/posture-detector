O Que É

O PostureAI é um sistema de detecção de postura em tempo real que monitora a posição do corpo enquanto você trabalha no computador. Todo o processamento ocorre diretamente no navegador via WebAssembly — nenhum dado de vídeo trafega pela rede, garantindo privacidade total.

Má postura prolongada pode causar dores cervicais, lombalgias e tensão muscular. O PostureAI age como um vigilante silencioso, alertando você antes que esses problemas se desenvolvam.

⚙

Como Funciona

  • 1
    Captura — O navegador acessa a webcam via getUserMedia. O vídeo nunca sai do dispositivo.
  • 2
    Detecção de Pose — MediaPipe PoseLandmarker identifica 33 pontos anatômicos por frame usando uma rede neural rodando em WebAssembly.
  • 3
    Análise de Ângulos — Dois ângulos são calculados: inclinação do pescoço (ombro→nariz vs. vertical) e inclinação do tronco (quadril→ombro vs. vertical).
  • 4
    Classificação — Postura "Ruim" quando pescoço > 40° e tronco > 10° por 5 frames consecutivos.
  • 5
    Alerta — Web Speech API emite voz em português, com cooldown de 30 segundos entre alertas.
✨

Por Que Foi Criado

Com o aumento do trabalho remoto e longas jornadas no computador, problemas posturais tornaram-se uma epidemia silenciosa. Mais de 80% dos trabalhadores de escritório relatam algum tipo de dor relacionada à má postura.

Este projeto entrega uma solução acessível que roda 100% localmente — sem envio de dados para nuvem, sem câmera acessada por terceiros, sem custo. O servidor Python serve apenas arquivos estáticos, sem tocar no vídeo.

// Modelos de Aprendizado de Máquina

Técnicas aplicadas: redes neurais, aprendizado supervisionado e não supervisionado

Supervisionado

MediaPipe PoseLandmarker

Arquitetura Rede Neural Convolucional (CNN) baseada em MobileNetV2 — encoder leve otimizado para dispositivos edge + decoder para regressão de coordenadas de landmarks
Tipo de Aprendizado Supervisionado — treinado com dataset de imagens anotadas manualmente com 33 landmarks corporais por humanos. Labels: coordenadas (x, y, z) + visibilidade de cada ponto anatômico
Input Frame de vídeo RGB (redimensionado para 224×224 internamente pelo runtime TFLite)
Output 33 landmarks × 4 valores = 132 floats por frame: coordenadas normalizadas (x, y, z ∈ [0,1]) + score de visibilidade
Features Usadas Landmarks selecionados: nariz (índice 0), ombro esquerdo (11), ombro direito (12), quadril esquerdo (23), quadril direito (24). Midpoints calculados para ombro e quadril → 2 ângulos derivados
Modelo pose_landmarker_lite.task — variante leve (~3MB), quantizada em float16. Roda via TFLite WASM no navegador sem GPU
Inferência 12 FPS cap — detectForVideo() com timestamp monotônico. Delegate CPU forçado para consistência cross-browser
Não Supervisionado

K-Means Clustering de Sessão

Algoritmo K-Means com K=3 clusters — implementado em JavaScript puro, sem dependências externas
Tipo de Aprendizado Não supervisionado — agrupa os dados da sessão sem rótulos pré-definidos, descobrindo padrões posturais específicos do usuário
Features (Input) Vetor 2D por amostra: [ângulo_pescoço, ângulo_tronco] — coletado 1x/segundo durante a sessão
Função de Distância
Distância Euclidiana: d = √((Δneck)² + (Δtorso)²)
Inicialização Centroides iniciais distribuídos uniformemente no dataset (simulando k-means++): centroide_i = amostra[i × (N/K)]
Convergência Iteração para quando nenhum ponto muda de cluster, ou após 60 iterações máximas. Reexecutado a cada 5 segundos conforme novos dados chegam
Output 3 clusters nomeados automaticamente por heurística nos centroides: "Ereta", "Pescoço Inclinado", "Tronco Curvado" — com proporção percentual de cada padrão na sessão
Mínimo de Dados Executado somente quando ≥ 15 amostras disponíveis (~15 segundos de sessão), garantindo pelo menos 5 pontos por cluster
